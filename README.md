# ГдеЗаправка — карта АЗС России

Веб-сервис с картой заправочных станций городов России. Водители могут отмечать:

- есть ли топливо (Есть / Лимит / Нет);
- какие марки доступны (АИ-92, АИ-95, АИ-98, АИ-100, ДТ, Газ);
- лимит на отпуск (в литрах);
- очередь на заправке;
- актуальные цены (₽/л).

Аналог сервиса [gdezapravka.ru](https://gdezapravka.ru/).

## Стек

- **Backend:** Node.js + Express + SQLite (better-sqlite3)
- **Frontend:** чистый HTML/CSS/JS
- **Карта:** Leaflet + OpenStreetMap (без API-ключей)

## Запуск

```bash
npm install
npm start
```

Откройте: http://localhost:3000

При первом запуске автоматически создаётся база `data.db` с демо-АЗС в 10 крупных городах.

## Импорт реальных АЗС (OpenStreetMap)

Демо-данные можно заменить/дополнить реальными АЗС из OpenStreetMap через Overpass API:

```bash
node import-osm.js "Екатеринбург"        # один город из списка
node import-osm.js all                     # все города из списка
node import-osm.js "Тюмень" 57.15 65.53    # произвольный город (координаты центра)
```

Данные © OpenStreetMap contributors (ODbL). Импортированные точки помечаются `source='osm'`
и не дублируются при повторном запуске (уникальный `osm_id`).

## Возможности

- Карта с цветными маркерами по статусу топлива (зелёный/жёлтый/красный/серый).
- Выбор города и фильтр по статусу.
- Список ближайших АЗС (сортировка по расстоянию при включённой геолокации).
- Карточка АЗС: статус, доступное топливо, цены, история отметок, кнопка «Маршрут».
- Форма «Отметить наличие» — без регистрации.
- Добавление цен водителями.
- **Рейтинг доверия (уверенность):** статус АЗС определяется не последней отметкой,
  а взвешенным консенсусом отметок за последние 12 часов. Вес каждой отметки =
  свежесть (полураспад 3 ч) × репутация автора. Показатель «Уверенность %» отражает
  согласованность и объём независимых отметок.
- **Антиспам:** анонимный `client_id` в cookie, ограничения на частоту:
  1 отметка на АЗС в 2 минуты от одного клиента, лимиты по клиенту и IP.
- **Репутация авторов:** растёт, когда отметки совпадают с итоговым консенсусом
  (влияет на вес будущих отметок), что снижает влияние спамеров и накрутки.

## API

| Метод | Путь                                      | Описание                                |
| ----- | ----------------------------------------- | --------------------------------------- |
| GET   | `/api/cities`                             | Список городов                          |
| GET   | `/api/stations?city=&near_lat=&near_lng=` | Список АЗС                              |
| GET   | `/api/stations/:id`                       | Детали АЗС (статус, цены, история)      |
| POST  | `/api/stations/:id/report`                | Отметить наличие                        |
| POST  | `/api/stations/:id/price`                 | Добавить цену                           |
| GET   | `/api/meta`                               | Справочники (статусы, топливо, очереди) |

## Автообновление данных (cron) и кластеризация

- **Автообновление OSM:** сервер сам обновляет АЗС из OpenStreetMap по расписанию.
  Интервал задаётся переменной окружения `OSM_REFRESH_HOURS` (по умолчанию 24 ч).
  `OSM_REFRESH_HOURS=0` — отключить. Есть ручной запуск: `POST /api/admin/refresh-osm`.
- **Кластеризация маркеров:** при большом числе АЗС маркеры группируются в цветные
  «облачка» (Leaflet.markercluster). Цвет облачка отражает преобладающий статус,
  число — количество АЗС в группе.
